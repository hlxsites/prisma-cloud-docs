=== Enhancements
[cols="50%a,50%a"]
|===

|*Feature*
|*Description*

|*Debian 13 Support*
//CWP-64272/CWP-64257

|Debian 13 (Trixie) the current stable distribution of Linux is now supported.

|*Support for TLS 1.3*
//CWP-64280

|TLS 1.3  the latest version of the Transport Layer Security protocol is now supported.

|*CIS Bottlerocket Benchmark Support*
//CWP-64282

|CIS Bottlerocket Benchmark control that gauges the compliance of the Bottlerocket container hosting operating system is now supported. Coverage for controls 1.3.1, 1.4.3, and 1.4.4 are now included.

=== Addressed Issues

[cols="50%a,50%a"]
|===

|*Issue ID*
|*Description*

// |CWP-64229
// |*Fixed Customer bug*:

// Description. 

|CWP-64402
|*Fix 502 Bad Gateway error in Runtime Security Module*

Resolved a Runtime Security connectivity issue.

|CWP-64398
|*Image Cleanup Logic Skips Images When Hosts Field is Empty*

Fixed an issue where images that have been deleted from their namespaces are still being shown in the console.

|CWP-64117
|*Cloud Discovery Limitation Resolved*

ECS Task Definition containers with launchType: External that are listed in the Inventory are not present in the Prisma Cloud Console ( Runtime Security > Radar). This issue is now fixed. 

|CWP-63717
|*Filepaths of secrets found by scanners other than agentless contain prefix of scan directory*

Resolved an issue with the Vulnerability scan report for registry images showing vulnerabilities that are tagged to the below image path which does not exist on the Image.

|CWP-64258
|*Cloud Radar Resource Reporting*

Fixed an issue where the reporting for Defended resources was sometimes incorrect. 

|CWP-64116
|*Console - host and serverless auto-defend rules are not filtered by custom role allowed cloud account id*

Resolved an issue with a custom role user not being able to access certain specifications when working with Defender Auto-Deploy Rules.

|CWP-64265
|*Macro fields not populated*

Fixed an issue where not all macro fields were populated when a Compliance trigger is set up to forward alerts via Webhook to Service Now.

|CWP-64459
|*NF Table Support*

Following the addition of  NF Tables support in WAAS, NF Tables support is also now available for Defender (CNNF and Runtime Policy).

*Note:* Open issues relating to the Compliance scan flow, will be addressed in a subsequent release.

|CWP-63255
|*Defender – Removed unused packages*

Removed multiple unused packages to reduce exposure to CVEs.
